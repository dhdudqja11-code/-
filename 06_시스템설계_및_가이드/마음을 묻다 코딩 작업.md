# 마음을 묻다 프론트엔드 작업 지시서 (V3: 프리미엄 웹 뷰어 모바일 최적화)

**대상:** GitHub Copilot (또는 Cursor AI)
**목표:** Next.js `page.tsx`의 동적 UI 및 모바일 UX 최적화(프리미엄 웹 뷰어) 완벽 구현

현재 백엔드(AI) 시스템은 `page.tsx`로 완벽한 다중 페이지용 JSON 배열을 보내주고 있습니다. 이제 프론트엔드에서 아래 3가지 작업을 수행하여 UI를 완성해야 합니다.

## 작업 1: 요금제별 동적 입력 폼 구현 (완료)
- Beta/Deep은 7문항 입력, Recovery는 7일 회복용 전용 문항 입력. (이미 코드에 적용됨)

## 작업 2: 7일 회복 편지용 다중 PDF DOM 분리 (완료)
- `html2canvas-pro`를 이용해 여러 장의 PDF를 인쇄하기 위해, 화면에 보이지 않는 `<div id="multi-page-pdf-root">` 내부에 `letterData.recovery_days` 배열을 돌면서 `.pdf-page` 클래스를 가진 여러 개의 A4 사이즈 div를 렌더링해야 합니다. (이미 코드에 적용됨)

## 작업 3: (핵심) 모바일 최적화 '프리미엄 웹 뷰어' 디자인 다듬기
현재 `page.tsx`에는 결제 완료 시 모바일 유저가 곧바로 편지를 읽을 수 있도록 화면에 렌더링하는 **'프리미엄 웹 뷰어'** 로직이 적용되어 있습니다. 코덱스는 아래 JSX 구조를 반드시 유지하면서, 폰트 크기, 여백, 모바일 반응형 디자인 등 미적 요소(CSS Tailwind)를 고급스럽게 다듬어 주세요.

### 📌 프리미엄 웹 뷰어 핵심 코드 구조 (page.tsx 818번째 줄 부근)
반드시 아래의 분기 로직과 코드를 유지해야 합니다. 코덱스는 코드를 복사한 뒤 스타일링만 개선하세요.

```tsx
{/* 결과 UI 분기 처리: 무료는 기존 엽서, 유료는 프리미엄 웹 뷰어 */}
{["free", "random"].includes(productType) ? (
  <div id="letter-container">
    {/* 기존 모눈종이 엽서 UI */}
  </div>
) : (
  <div className="w-full flex flex-col items-center animate-fade-in mb-8">
    {/* 프리미엄 웹 뷰어 컨테이너 (이 부분의 디자인을 고급스럽게 유지 및 발전시켜 주세요) */}
    <div className="w-full max-w-2xl bg-[#FDFBF7] rounded-3xl p-8 md:p-16 border border-slate-200 shadow-layered mb-8 text-left">
      <div className="text-center mb-16 border-b border-slate-200 pb-12">
        <h1 className="text-3xl md:text-4xl font-serif text-slate-800 mb-4">{letterData?.cover?.title || "당신을 위한 문장 처방전"}</h1>
        <p className="text-lg text-slate-500 font-serif">{letterData?.cover?.heart_name}</p>
      </div>

      {(!letterData?.recovery_days || letterData.recovery_days.length === 0) ? (
        // Beta, Deep, Gift 요금제 렌더링
        <div className="font-serif leading-loose text-lg text-slate-700">
          {(letterData?.page_letter_paragraphs?.length ? letterData.page_letter_paragraphs : [letterData?.letter || ""]).map((para, idx) => (
            <p key={`web-para-${idx}`} className="mb-8">{para}</p>
          ))}
          
          <div className="mt-16 bg-white p-8 rounded-2xl border border-amber-100/50 shadow-inner">
            <h3 className="text-xl mb-6 font-serif text-slate-800 font-bold border-b border-amber-100 pb-2 inline-block">오래 간직할 문장</h3>
            {letterData?.page_sentences?.map((sentence, idx) => (
              <p key={`web-s-${idx}`} className="mb-4 text-slate-600">"{sentence}"</p>
            ))}
            
            <h3 className="text-xl mt-12 mb-6 font-serif text-slate-800 font-bold border-b border-amber-100 pb-2 inline-block">나에게 묻는 질문</h3>
            {letterData?.page_questions?.map((q, idx) => (
              <p key={`web-q-${idx}`} className="mb-4 text-slate-600">Q. {q}</p>
            ))}

            {letterData?.page_action && (
              <>
                <h3 className="text-xl mt-12 mb-6 font-serif text-slate-800 font-bold border-b border-amber-100 pb-2 inline-block">오늘의 작은 행동</h3>
                <p className="text-slate-600">{letterData.page_action}</p>
              </>
            )}
          </div>
        </div>
      ) : (
        // Recovery (7일 회복) 요금제 렌더링
        <div className="font-serif text-slate-700">
          {letterData.recovery_days.map((dayData, idx) => (
            <div key={`web-recovery-${idx}`} className="mb-20 last:mb-0">
              <h2 className="text-2xl font-serif text-slate-800 mb-8 border-b border-slate-200 pb-2 inline-block">{dayData.day}일차 회복 편지</h2>
              <p className="leading-loose text-lg whitespace-pre-wrap mb-10">{dayData.letter}</p>
              <div className="bg-white p-6 rounded-2xl border border-slate-100 shadow-sm">
                <h3 className="text-lg font-serif text-slate-800 mb-3 font-bold">오늘의 작은 행동</h3>
                <p className="text-slate-600">{dayData.action}</p>
              </div>
            </div>
          ))}
        </div>
      )}
    </div>
    
    {/* PDF 영구 소장 다운로드 버튼 */}
    <button
      id="download-pdf-btn"
      onClick={handleDownloadPDF}
      className="px-10 py-4 bg-slate-900 text-white rounded-full font-serif text-lg tracking-wide hover:bg-slate-800 transition shadow-xl flex items-center gap-3 animate-fade-in z-20 active:scale-95 duration-150 cursor-pointer"
    >
      <svg className="w-5 h-5" fill="none" stroke="currentColor" viewBox="0 0 24 24">
        <path strokeLinecap="round" strokeLinejoin="round" strokeWidth="2" d="M4 16v1a3 3 0 003 3h10a3 3 0 003-3v-1m-4-4l-4 4m0 0l-4-4m4 4V4" />
      </svg>
      이 처방전 영구 소장하기 (PDF 다운로드)
    </button>
  </div>
)}
```


## 작업 4: 글로벌 다국어(i18n) 번역 딕셔너리(Dictionary) 구현
현재 백엔드(AI)는 프론트엔드로부터 전달받는 `language` 파라미터를 통해 편지 본문을 사용자의 국가 언어로 즉시 번역해서 보내도록 완벽히 수술되었습니다.
이제 껍데기 UI(버튼, 폼 설명 등)를 다국어로 렌더링할 차례입니다.

1. `page.tsx` 내에 `translations` 객체를 만드세요.
   ```tsx
   const translations = {
     ko: { selectTier: "어떤 처방을 원하시나요?", back: "뒤로 가기", ... },

   };
   // 기존에 있는 isKorean 상태를 활용: const t = isKorean ? translations.ko : translations.en;
   ```
