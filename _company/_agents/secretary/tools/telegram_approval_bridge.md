# 📱 모바일 양방향 텔레그램 승인 브릿지

비서(Secretary)가 사장님(사용자)의 모바일 텔레그램 앱으로 중요 결재(배포, 메일 발송 등) 요청 카드를 송신하고, 사장님이 실시간으로 모바일에서 직접 `승인 👍` 또는 `반려 ❌` 버튼을 눌러 피드백을 전달할 때까지 안전하게 대기(롱 폴링)하여 그 결과를 시스템에 반영해주는 양방향 승인 중개 도구입니다.

---

## 💡 어떻게 도와주나요?

- **사장님 전용 결재 카드 발송**: 텔레그램의 Inline Keyboard(인터랙티브 버튼) 기능을 사용하여 모바일 앱에서 한 번의 터치로 승인/반려할 수 있는 직관적인 카드를 전송합니다.
- **실시간 로컬 롱 폴링**: ngrok 등 외부 터널링 툴이나 사설 IP 환경 제약 없이, 텔레그램의 `getUpdates` API를 활용하여 100% 안전하게 사장님의 클릭 신호를 추적합니다.
- **동적 비주얼 피드백**: 사장님이 버튼을 누르면 봇 메시지 본문이 즉각 **"🎉 [결재 승인 완료]"** 혹은 **"❌ [결재 최종 반려]"** 상태로 동적으로 편집되어, 터치 완료 여부를 모바일에서 즉시 확인할 수 있습니다.
- **협업 자동화 연계**: 결재 결과가 나면 `_company/approvals/<action_id>.json` 파일에 상태를 안전하게 기록하고 프로세스 반환 코드(Exit Code)를 전달하므로, 개발자(코다리)나 다른 에이전트의 배포 파이프라인과 완벽히 연동됩니다.

---

## ⚙️ 입력 매개변수 (Arguments)

| 파라미터 | 기본값 | 설명 |
| :--- | :--- | :--- |
| `--action-id` | `test_action` | 결재 프로세스를 식별할 고유 고유명사 ID (예: `deploy_v1.0.0`) |
| `--title` | `테스트 결재 요청` | 사장님 모바일에 표시될 굵은 헤더 제목 |
| `--description` | (예시) | 구체적으로 어떤 작업에 대한 승인을 얻고자 하는지에 대한 본문 요약 |
| `--timeout` | `300` | 승인을 대기할 최대 제한 시간 (단위: 초, 기본 5분) |
| `--test` | `False` | 모바일 승인 카드 및 버튼 감시가 잘 구동되는지 즉시 2분간 자가 검증하는 테스트 모드 구동 |

---

## 🛠️ 영숙이(비서)의 자율 행동 지침 (Agent Guideline)

1. **자동 승인 배제**: 배포(`deploy`), 유튜브 업로드(`publish`), 외부 메일 발송(`send_email`) 등 외부에 중대 영향을 미치는 행동을 개시하기 전에 반드시 이 브릿지 툴을 먼저 작동시켜 사장님의 최종 결재를 획득해야 합니다.
2. **반려 시 행동**: 결재가 반려되거나(Exit Code 2), 타임아웃으로 만료된 경우 관련 작업을 즉시 중단하고 반려 사유 또는 중단 상태를 메모리에 기록한 뒤 정중하게 보고해야 합니다.
3. **로깅**: 모든 결재 요청과 그 피드백 결과는 `_company/approvals/` 디렉터리에 JSON 이력으로 누적 보관되어 차후 ROI 및 손실 회피 비용 산정의 기초 데이터로 활용됩니다.

---

## 🔒 보안 및 신뢰성

- 이 도구는 `config.md`에 입력된 텔레그램 자격증명을 공유하며, 기밀 데이터는 외부로 절대 유출되지 않습니다.
- 사장님이 버튼을 여러 번 누르더라도, 봇 서버의 안전한 중복 응답 방지(offset) 로직으로 인해 최초 1회 입력만 유효하게 처리됩니다.
