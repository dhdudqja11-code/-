## 🚨 Mini ROI 시뮬레이터: 사용량 제한 경고 시스템 (UX/UI Mockup)

**톤앤매너:** **신뢰 기반의 위기감(Authoritative Urgency)**
*   **메인 컬러 (Trust):** `#0A1931` (딥 네이비 블루 - 밤하늘, 전문성)
*   **보조 컬러 (Data/Neutral):** `#F4F7FA` (밝은 회색-흰색, 명료함)
*   **위기 경고 (Warning):** 오렌지 계열 (`#FF9800`) $\rightarrow$ 빨강 계열 (`#D32F2F`)

### 1. [주요 화면] 대시보드 상단 위젯 (Primary Widget Display)

시스템에 접속하는 순간, 사용자가 가장 먼저 마주하게 되는 핵심 정보입니다. 경고가 발생하면 이 위젯 전체의 디자인이 변해야 합니다.

| 요소 | 상태: 정상 작동 (Safe) | 상태: 임계값 접근 (Warning/Orange) | 상태: 쿼터 초과 (Critical/Red) |
| :--- | :--- | :--- | :--- |
| **배경** | `#E6F0FF` (밝은 네이비 톤) | 배경색 변화 없음. 경고 요소 강조. | `#FFF3E0` (약간의 위험 노란 기운 부여) |
| **제목** | `[Status] API 사용 현황: 안정적입니다.` | `⚠️ [Warning] 할당량 임계치 도달 임박` | `🛑 [CRITICAL] 서비스 일시 중단 위험 발생!` |
| **주요 시각 요소 (Quota Bar)** | 🟢 막대 그래프가 녹색(Safe Green)을 유지하며, 남아있는 양이 명확히 표시. | 🟠 막대 그래프의 색상이 오렌지 계열로 점진적으로 변화하며, 감소 추세를 보여줌. | 🔴 막대 그래프 전체가 강렬한 빨간색으로 채워지고, "0%"가 깜빡임. |
| **핵심 데이터** | `잔여 쿼터: 125/300` / `사용률: 41%` | `남은 쿼터: 8/300` $\rightarrow$ *경고!* `/ 예상 소진 시점: 남은 세션 기준 3시간 후` | `잔여 쿼터: 0 / 300` $\rightarrow$ **[서비스 사용 불가]** |

### 2. [상호작용] 경고 모달 (The Explanation Modal)

사용자가 위젯에서 경고(Orange/Red)를 인지하고 어떤 행동을 취해야 하는지 명확히 설명하는 팝업 창입니다. 이 부분이 '안전장치 작동 원리'의 핵심입니다.

**[모달 제목] : [Mini ROI 시스템] 사용량 제한 알림: 규제 준수 게이트웨이 경고**
*(부제목: 현재 데이터 패턴을 분석한 결과, 할당량을 초과할 위험에 처했습니다. 아래 조치를 통해 리스크를 방어하세요.)*

#### 🖼️ 모달 구조 및 내용

1.  **위험 시각화 (Top Section):**
    *   **요소:** **실시간 경고 그래프(The Downward Slope Graph)**
    *   **디자인:** Y축은 '남은 쿼터', X축은 '경과 시간'. 현재 지점(사용자)이 급격하게 하락하는 추세가 빨간색 선으로 표시되고, 이 아래에 **'임계값 (Threshold Line)'**이라는 수평선(Orange)을 그려놓습니다.
    *   **메시지:** "현재 사용 패턴은 임계값을 3시간 내로 돌파할 것으로 예측됩니다."

2.  **위험 원인 분석 (Middle Section - The Why):**
    *   **제목:** **⚠️ 위험 진단: 왜 경고가 발생했나요?**
    *   **내용:** "최근 시간 동안 [API 호출 횟수] 증가, 또는 [특정 기능]의 과도한 사용 패턴이 감지되었습니다. 이는 시스템 자원의 비정상적 소모를 유발합니다." (구체적인 원인을 명시)

3.  **시스템 작동 원리 설명 (Bottom Section - The How):**
    *   **제목:** **🛡️ 방어 장치 가동 원리: 규제 준수 아키텍처가 보호합니다.**
    *   **핵심 텍스트 (권위적 어투 유지):** "본 시스템은 단순 차단이 아닌, **'감사 가능한 제어(Audit-able Control)'**를 통해 서비스를 보호합니다. 임계값 도달 시, 다음 단계의 안전장치가 작동하여 비즈니스 연속성을 보장합니다."
    *   **단계별 설명 (Flowchart):**
        1.  **(Current State)** $\rightarrow$ 사용량 임계치 초과 감지
        2.  **(Action 1)** $\rightarrow$ API 게이트웨이에서 트래픽을 **제한(Rate Limiting)**하고, 사용자에게 경고를 전송합니다.
        3.  **(Action 2)** $\rightarrow$ 모든 요청은 **불변 감사 로그**에 기록되어 법적 증거로 남습니다. (가장 중요한 문구)
        4.  **(Result)** $\rightarrow$ 시스템 전체의 안정성을 유지하며, 고객사에게는 **'규제 준수 보호'**라는 가치를 제공합니다.

4.  **해결책 및 CTA (Action Block):**
    *   **제목:** **✅ 지금 당장 취해야 할 조치:**
    *   **옵션 1 (가장 강력한 권유):** "추가 사용이 필요하시면, [Assurance Pro 플랜]으로 업그레이드하여 리스크를 즉시 해소하세요." $\rightarrow$ *CTA 버튼 (Orange)*
    *   **옵션 2:** "불필요한 호출을 줄여주세요. 현재 세션에서 분석할 데이터를 재검토해 주세요." $\rightarrow$ *CTA 링크 (Neutral)*
