# 📑 [Writer] Assurance Value Module: 서사 구조화 카피라이팅 가이드라인 v1.0

## 🎯 목표: 스토리텔링을 데이터 포인트로 전환
이 모듈은 사용자가 느끼는 막연한 불안감(Pain)을 구체적인 비즈니스 위험으로 정의하고, 이 위험을 해소하는 과정(Assurance) 자체를 핵심 판매 가치로 포지셔닝합니다.

### 1. 구조화된 서사 매핑 (The Golden Triad Schema)
모든 카피는 다음의 세 가지 단계적 요소로 분리되어야 하며, 각 단계별 키워드와 논리를 정의하여 API 입력값으로 활용할 수 있도록 구성해야 합니다.

| 단계 | 역할 및 목표 | 내부 로직/API 대응 | 핵심 질문 (Copywriter Check) |
| :--- | :--- | :--- | :--- |
| **[Pain Description]** | 현재 상태의 위험 인지. 감정적 공포를 구체적인 *재무적 손실*로 치환한다. (Why they are scared?) | `loss_data` 입력값 기반: Risk Category, Estimated Loss Value | "지금 당장 돈/시간을 잃는다면 무엇일까?" |
| **[Trigger Event]** | 위협이 현실화되는 '계기' 또는 '순간의 깨달음'. 문제 제기의 임계점. (The moment of crisis.) | `user_purpose_narrative` 내 스토리 흐름 정의 | "무엇을 보거나 듣는 순간, '위험하다'고 느낄까?" |
| **[Assurance Statement]** | 우리의 솔루션을 통해 확보하게 되는 *명확한 통제력*과 *미래의 확신*. (The definitive solution.) | `calculate-assurance` 최종 Output: Certainty Index. | "이것을 사용하면, 더 이상 아무것도 걱정하지 않아도 될까?" |

### 2. 콘텐츠 자동 생성 파이프라인용 템플릿 및 예시
아래는 각 산업별로 적용 가능한 구체적인 시나리오와 그에 따른 구조화된 카피 아웃풋입니다.

#### 💡 [Scenario A] 규제 준수 리스크 (헬스케어/금융)
*   **분야:** 의료 기기 인증 및 데이터 프라이버시(HIPAA, GDPR 등).
*   **핵심 Pain:** 법규 변경 주기를 따라잡지 못하여 발생하는 최대 벌금 및 영업 정지 위험.

| 구성 요소 | 구조화된 카피/텍스트 (API 입력용) | 논리적 흐름 설명 |
| :--- | :--- | :--- |
| **[Pain Description]** | *제목:* 2024년 개정되는 데이터 프라이버시 법규. 미준수 시, 국내 최대 벌금 OO억 원의 직접적인 재무 위험에 노출됩니다. (Risk Category: 규제 위반 / Loss Value: OOO원) | 감성적 불안 $\to$ 수치적 공포로 치환하여 즉각적인 위협을 강조. |
| **[Trigger Event]** | "최근 경쟁사가 동일한 규정 미준수로 인해 영업 정지 명령을 받은 사례"를 보며, 우리도 언제든 같은 위험에 처할 수 있음을 깨닫는 순간. | 외부 자극(뉴스/사례)이 내재된 두려움을 폭발시키는 트리거 역할 수행. |
| **[Assurance Statement]** | 당사의 자동화 모듈은 법규 변화를 실시간으로 추적하고, 선제적으로 시스템에 반영하여 *'미준수 리스크 0%'* 상태를 유지하는 확신(Certainty)을 제공합니다. | 문제 해결이 '쉬움'이 아니라 '근본적인 무위험 상태 확보'임을 강조. |

#### 💡 [Scenario B] 운영 효율성 및 확장성 부족 (제조/공급망 관리)
*   **분야:** 공급망의 비효율적 단절 지점(Bottleneck)으로 인한 생산 중단 위험.
*   **핵심 Pain:** 특정 부품이나 인력에 대한 의존도가 높아, 작은 문제로도 전체 사업이 마비될 수 있음.

| 구성 요소 | 구조화된 카피/텍스트 (API 입력용) | 논리적 흐름 설명 |
| :--- | :--- | :--- |
| **[Pain Description]** | 특정 공급처의 지연 사태(Lead Time Delay) 한 번으로, 전체 생산 라인 가동 중단에 따른 월평균 OO억 원 규모의 매출 손실이 발생할 수 있습니다. (Risk Category: 운영 리스크 / Loss Value: OOO원) | 비전통적 위험을 '매출 감소'라는 가장 확실한 지표로 연결. |
| **[Trigger Event]** | 핵심 공급처에서 갑작스러운 재고 부족 경고가 뜨거나, 물류 경로에 예측 불가능한 지연이 발생했을 때 느끼는 통제력 상실의 순간. | '예측 가능했던 것이 깨지는' 순간의 무력감을 활용하여 공감 유도. |
| **[Assurance Statement]** | 시스템은 다중 공급망 시뮬레이션과 실시간 대체 경로 분석을 통해, 단일 실패 지점(Single Point of Failure) 리스크를 완전히 제거하고 *최적화된 대안 경로*를 자동으로 제시합니다. | 단순히 문제를 해결하는 것을 넘어, '모든 경우의 수를 관리한다'는 포괄적 통제력을 강조. |

#### 💡 [Scenario C] 내부 프로세스 비효율 (마케팅/HR)
*   **분야:** 수작업 의존성이 높은 업무 과정(Manual Process Dependency).
*   **핵심 Pain:** 직원들의 시간 낭비가 누적되어, 핵심 인재들이 본질적인 가치 창출에 집중하지 못하는 상황.

| 구성 요소 | 구조화된 카피/텍스트 (API 입력용) | 논리적 흐름 설명 |
| :--- | :--- | :--- |
| **[Pain Description]** | 매주 반복되는 보고서 취합 및 데이터 정리 과정에 직원 1인당 월 평균 OO시간을 소요하고 있으며, 이는 핵심 전략 수립 시간이 될 수 있는 기회비용 손실입니다. (Risk Category: 인력 비효율 / Loss Value: OOO원) | 돈 대신 '가장 귀한 자산(시간/인재)'의 손실로 Pain 포지셔닝. |
| **[Trigger Event]** | 중요한 의사결정 회의 직전에, 아직도 수많은 사람이 엑셀 시트와 복잡한 보고서를 들고 마주 앉아 시간을 낭비하는 장면을 목격했을 때 느끼는 답답함과 비효율성의 깨달음. | '지루함'이나 '비합리성'을 위협으로 승화시켜 공감대를 형성. |
| **[Assurance Statement]** | AI 기반의 자동 워크플로우 설계와 통합 플랫폼은, 수작업 과정을 제로(Zero)에 가깝게 줄이고, 모든 직원이 가장 창의적이고 고차원적인 업무에 집중할 수 있도록 시간적 자율성을 확보합니다. | '노동력 절감'이 아닌 '인재 역량 극대화 및 자유'라는 가치로 승격. |
