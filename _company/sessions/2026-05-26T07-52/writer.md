### 🚨 Failure Message A: 인증 실패 (Authentication Failure)
*(사용자가 접근 권한/자격 증명에 실패했을 때)*
**목표:** 사용자의 실수로 돌리지 않고, '시스템의 보안 프로토콜이 작동했다'는 점을 강조하여 신뢰도를 높인다.

#### 💡 버전 1: [법적 무결성 강조] - (가장 권위적)
*   **헤드라인/제목:** `[ACCESS DENIED] : 접근 자격 검증 실패. 보안 프로토콜 작동.`
*   **본문 설명:** "요청하신 리소스에 대한 인증 정보가 현재의 보안 매개변수와 불일치합니다. 이 로그는 시스템이 무단 접근 시도를 감지하고, 귀하의 데이터 건전성을 최우선으로 보호했음을 의미합니다."
*   **해결책/CTA:** "접속을 재시도하거나 [계정 복구 요청] 버튼을 눌러 보안 절차를 완료해 주십시오. 모든 기록은 Audit Block에 남습니다."

#### 💡 버전 2: [리스크 경고 강조] - (긴급성 부여)
*   **헤드라인/제목:** `[SECURITY ALERT] : 인증 프로토콜 위반 감지. 즉시 조치 필요.`
*   **본문 설명:** "시스템은 외부 또는 내부의 잠재적 접근 위험을 포착하고 현재 접속을 일시 중단했습니다. 이는 귀하의 핵심 자산을 보호하기 위한 필수적인 방어 메커니즘입니다."
*   **해결책/CTA:** "안전한 재접속을 위해 [2FA 재설정] 절차를 거쳐 주십시오. 보안 강화가 완료되면 즉시 접근이 허용됩니다."

#### 💡 버전 3: [기술적 설명 강조] - (전문성 어필)
*   **헤드라인/제목:** `[AUTH PROTOCOL ERROR] : 유효 토큰 만료 또는 불일치.`
*   **본문 설명:** "현재 세션의 JWT(JSON Web Token)가 만료되었거나, 시스템이 정의한 보안 정책과 상충하는 자격 증명을 사용했습니다. 데이터 무결성을 위해 연결을 끊었습니다."
*   **해결책/CTA:** "페이지를 새로고침 하거나 [재로그인] 버튼을 클릭하여 새로운 유효 세션을 확립해 주십시오."

### ✅ Success Message B: 감사 기록 생성 완료 (Audit Block Generated)
*(위험 트랜잭션이나 중요한 활동이 성공적으로 처리되었으나, 그 과정 자체가 법적 증거로 남았음을 알릴 때)*
**목표:** 사용자가 단순한 '성공'을 넘어, '법적/재무적 책임이 부여된 기록'이 생성되었다는 점을 인식하게 한다.

#### 💎 버전 1: [법적 책임 강조] - (가장 중요하고 무게감 있는 어조)
*   **헤드라인/제목:** `[AUDIT COMPLETE] : 불변 감사 기록(Audit Block)이 성공적으로 확정되었습니다.`
*   **본문 설명:** "요청하신 트랜잭션은 시스템의 엄격한 검증을 거쳐, 위·변조가 원천 차단되는 블록체인 기반의 '감사 기록'으로 영구 보존되었습니다. 모든 행위는 이제 법적 증거로 기능합니다."
*   **추가 안내:** "생성된 Audit Block ID: [XXXX-YYYY]. 이 기록은 언제든 추적 가능하며, 귀사의 핵심 리스크 관리 자료입니다."

#### 💎 버전 2: [정보의 권위 강조] - (전문적인 설명)
*   **헤드라인/제목:** `[DATA INTEGRITY CONFIRMED] : 트랜잭션 로그가 Chain-Hashed 블록에 기록되었습니다.`
*   **본문 설명:** "처리된 모든 데이터는 체이닝 해싱(Chaining Hashing) 구조를 통해 이전 기록과 연결되어 무결성이 확보되었습니다. 이는 단순한 '기록'을 넘어선 '증명 가능한 사실'입니다."
*   **추가 안내:** "자세한 감사 내역은 [감사 로그 조회] 메뉴에서 확인 가능하며, 데이터의 출처와 시간적 흐름이 완벽히 보존되어 있습니다."

#### 💎 버전 3: [사용자 가치 강조] - (결과물에 초점)
*   **헤드라인/제목:** `[RISK MITIGATED] : 핵심 위험 트랜잭션 기록 완료.`
*   **본문 설명:** "귀하의 중요한 데이터 흐름(예: 계약 체결, 자금 이동)이 시스템 내부에서 최고 수준의 법적 무결성으로 처리되었습니다. 이제 이 과정 전체가 완벽하게 증명됩니다."
*   **추가 안내:** "지금 바로 [ALV 리스크 대시보드]로 이동하여 방금 기록된 데이터를 기반으로 실시간 재무 손실 회복력을 평가해 보십시오."
