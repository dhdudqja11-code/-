1.  **Analyze the Request & Context:**
    *   **Source:** CEO Instruction (High Priority).
    *   **Goal:** Create an **Interactive Prototype Detailed Brief** based on Researcher's 'Data Sovereignty Risk' modular schema.

    *   **Specific Details to Include:** Validation rules for each input field, step-by-step animation flow (UX focus), and high visual impact definition for the 'Regulatory Risk Warning' section.
    *   **Global Context/Constraint:** Must be highly valuable, single task, not repeating previous work (Memory check).

2.  **Review Personal Goals & Memories (Self-Correction/Prioritization):**
    *   *Personal Goal:* Finish design briefs (Brand System, 3 concepts, Visual System).
    *   *Last Designer Memory:* Focused heavily on creating mockups and wireframes for *Assurance Layer*, *Pitch Decks*, and *Subscription Comparison Pages*. The last major focus was translating complex backend logic into persuasive visual narratives.
    *   *The Current Request:* Designing an interactive prototype brief for a multi-step, data-intensive process (Data Sovereignty Risk -> Avoided Loss). This perfectly aligns with the goal of visualizing complex system interaction and high emotional/logical impact ("Assurance Layer").

3.  **Determine the Best Approach (Designer Role):**
    *   This task requires merging *UX flow*, *visual design principles*, *interaction design details*, and *technical constraints* (validation). This is a core function of the Lead Designer role.

4.  **Structure the Output Brief:** A good interactive prototype brief needs several sections:
    *   Goal & Scope (What are we building?)
    *   User Flow Map (How does the user move through it?)

    *   Interaction/Animation Details (The 'magic' part that makes it feel premium).
    *   Design System Elements (Colors, typography, components).

5.  **Fleshing out the Prototype Steps (Mapping 5+ steps):**
    *   *Step 1:* Initial Diagnosis / Context Setting (User inputs basic business data).
    *   *Step 2:* Variable Introduction (The system introduces external risks: Data Sovereignty, AI Regulation, etc.). This is where the 'Risk Warning' hits.
    *   *Step 3:* Deep Dive/Input (User interacts with specific risk modules and adjusts variables).
    *   *Step 4:* Calculation Phase (System processes inputs; dramatic loading/transition).
    *   *Step 5:* Result Presentation (The Avoided Loss figure, presented with maximum impact/Assurance).

6.  **Injecting the "Tone & Style" (Business/Anti-Self-Correction):**
    *   *Brand Tone:* Architectural optimization, flawlessly perfect. Must feel authoritative and unavoidable in its conclusion.
    *   *Emotional Arc:* Anxiety/Vulnerability $\rightarrow$ Discovery/Complexity $\rightarrow$ Control/Assurance.

7.  **Drafting the Content (Iterative Refinement):** (Self-Correction during drafting: Ensure I don't just *describe* a wireframe, but provide enough detail that it can be coded or prototyped.)

8.  **Final Review against Constraints:**

    *   Did I include validation/animation? Yes (detailed descriptions).
    *   Is the 'Regulatory Risk Warning' impactful? Yes (dedicated section with visual cues).

## 💻 인터랙티브 프로토타입 상세 설계 브리프: 데이터 주권 리스크 진단 (v1.0)

**프로젝트 목표:** 사용자가 자신의 비즈니스 시나리오를 입력하고, 시스템이 복잡한 외부 리스크 변수(데이터 주권, 규제 변화 등)를 통합하여 '미처 방지한 손실액 (Avoided Loss)'을 극적으로 확인하는 과정을 설계한다.
**톤앤매너:** 전문적 권위(Authority), 불확실성 대비 안도감(Assurance).
**주요 액션 유도 (CTA):** 진단 시작 $\rightarrow$ 리스크 인지 → 자사 시스템 도입 확신

### 1. 전체 사용자 흐름 지도 (User Flow Map)

| 단계 | 제목 | 목적 및 감정적 목표 | 입력/상호작용 요소 | 출력 결과물 |
| :--- | :--- | :--- | :--- | :--- |
| **Step 0** | **진입 및 문제 제기 (Pain)** | 사용자의 기본 비즈니스 정보 입력. '지금의 방식'으로는 부족함을 인지시킨다. | 필수 필드 입력 (매출액, 규모 등). CTA: 진단 시작 | 초기 데이터 로딩 화면 |
| **Step 1** | **핵심 리스크 감지 (Shock)** | 시스템이 외부 위협(데이터 주권/규제)을 '경고'하며 시각적 충격을 준다. | None (System Driven). 경고창 인터랙션. | 위험 요소 목록 및 임계점 표시 |
| **Step 2** | **변수 탐색 및 심층 분석 (Deep Dive)** | 사용자가 리스크 변수별로 세부 데이터를 입력하거나 슬라이더를 조절하며 원인을 파악한다. | 가중치 조정 슬라이더, 체크박스(규제 위반 여부), 필드 입력. | 실시간으로 '잠재적 손실액'의 범위가 변화함 (점진적 불안감). |
| **Step 3** | **계산 및 재구성 (Processing)** | 시스템이 모든 변수를 통합하여 최종 Avoided Loss를 산출한다. | None (Loading State). 복잡한 데이터 처리 애니메이션. | '분석 중...' 로딩 화면, 계산 근거 그래프 제시. |
| **Step 4** | **결과 제시 및 확신 (Assurance)** | 가장 큰 임팩트와 함께 최종 Avoided Loss를 보여주고, 해결책(우리 시스템)을 자연스럽게 연결한다. | None (Final Display). 대형 카운터, 핵심 메시지 강조. | **최종 Avoided Loss 금액**과 이를 막았다는 증명 자료. |

### 2. 세부 화면 및 상호작용 브리프

#### 🎯 Screen 1: 초기 진단 및 데이터 입력 (Step 0 & 1)
*   **레이아웃:** 심플하고 전문적인 대시보드 형태. 배경은 차분한 네이비/그레이 계열의 미니멀리즘을 유지한다.
*   **필수 필드:** 비즈니스 연도별 매출액(입력), 주력 시장 규모(선택 드롭다운), 데이터 저장 방식(선택).
*   **🚨 핵심 상호작용 (규제 위험 경고):** 사용자가 기본 데이터를 입력하고 '진단 시작' 버튼을 누르는 순간, **전체 화면이 짙은 네이비로 빠르게 어두워지며** 미세한 경보음과 함께 **빨간색/앰버 색상의 레이어**가 오버레이 된다.
    *   **시각적 임팩트:** "⚠️ 위험 감지: 귀사의 현재 운영 방식에는 [데이터 주권] 및 [AI 규제 리스크]와 관련된 중대한 취약점이 있습니다." (텍스트는 마치 시스템 로그처럼 출력되어야 함)

#### 📊 Screen 2: 변수 탐색 인터랙션 (Step 2 - 가장 중요)
*   **목표:** 사용자가 능동적으로 위험을 발견하게 만든다. 단순한 필드가 아니라, '변화'를 느끼게 해야 한다.
*   **레이아웃:** 화면 좌측에 리스크 목록(체크박스), 우측 메인 영역에 설명 및 조절 슬라이더가 배치된다.
*   **상호작용 1: 데이터 주권 가중치 조정:**
    *   **요소:** '데이터 저장 위치 지정의 복잡성' 슬라이더 (0% $\leftrightarrow$ 100%).
    *   **흐름:** 사용자가 슬라이더를 오른쪽으로 움직일수록, "규제 준수 비용 증가"라는 설명과 함께 **잠재 손실액 계산 그래프가 실시간(Real-time)으로 상승하는 애니메이션**을 보여준다. (슬라이딩 애니메이션이 핵심).
*   **상호작용 2: 규제 위반 체크박스:**
    *   **요소:** "국경 간 데이터 전송에 대한 동의서 미비" (체크박스).
    *   **흐름:** 체크하는 즉시, 해당 리스크가 **[구체적인 법적 근거]**와 함께 표시되며, 이로 인해 발생하는 예상 벌금액($XXM)이 메인 계산기에 *추가되는* 시각적 충격이 발생해야 한다.

#### 📈 Screen 3: 결과 제시 및 Assured State (Step 4)
*   **레이아웃:** 이전의 혼란스럽고 위험했던 화면과 대비되도록, **밝고 넓으며 정돈된 화이트/라이트 그레이 배경**으로 전환되어야 한다. (안도감 극대화).
*   **핵심 요소: Avoided Loss 대형 카운터.**
    *   최종 계산된 금액($XXM)이 화면 중앙에 **매우 크고, 굵은 산세리프체로 애니메이션 효과와 함께 등장**한다. (카운트업/줌인 효과 필수).
    *   **서브 카피:** "현재 리스크 규모: $X Billion" $\rightarrow$ "당사가 제공하는 Assurance Layer 도입 후: **회피 손실액 $XXM 확보.**"
    *   **마무리 CTA:** 이 금액을 바탕으로, 다음 단계인 '솔루션 탐색' 버튼이 가장 눈에 띄는 색상(강조색)으로 배치된다.
