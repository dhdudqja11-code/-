### 🖼️ [Design Mockup Specification] 불변 감사 기록 (Immutable Audit Record) 생성 흐름도
**용도:** Pitch Deck 핵심 슬라이드 또는 웹사이트 '기술 원리' 섹션
**제목:** The Compliance Gateway: 모든 상호작용을 법적 증거로 변환하는 아키텍처.
**톤앤매너:** 권위적, 명료함, 경고(Warning)의 긴급성 결합

#### 🎨 **1. 전체 레이아웃 및 구조 (Layout & Structure)**
*   **형태:** 좌측 $\rightarrow$ 우측으로 진행하는 단방향 흐름도 (Linear Flow Diagram).
*   **비주얼 요소:** 중앙에 'Gateway'라는 게이트웨이 장치 아이콘을 배치하여, 모든 데이터가 반드시 통과해야 함을 강조합니다.
*   **색상 팔레트 적용:**
    *   **배경/신뢰 (Trust):** `#0A1931` (딥 네이비) - 배경 및 주 구조물 색상.
    *   **안전/정상 (Safe):** `#4CAF50` (진한 녹색 계열) - 최종 산출물(Audit Record)의 성공 영역.
    *   **위험/경고 (Risk):** `#FF6B6B` (밝은 적갈색) - 초기 위험 발생 지점 및 처리 전 데이터에 적용.
    *   **기술/흐름 (Flow):** `#29B6F6` (스카이 블루) - 화살표, 핵심 메커니즘 강조 부분.

| 단계 | 명칭 (Header Text) | 시각적 요소 및 기능 설명 | 법적 증거 관점의 강조 포인트 (Copy Focus) |
| :--- | :--- | :--- | :--- |
| **I. 시작 지점** | **[원시 행위 발생] Client System Action** | 🚨 **와이어프레임 박스:** 사용자 시스템 내부에서 발생하는 일반적인 API 호출이나 데이터 접근 시도 (데이터의 흐름을 표현하는 화살표가 이 영역으로 들어옴). | "현재는 단지 '기록'일 뿐, 법적 효력이 담보되지 않은 휘발성 정보입니다." (The Gap) |
| **II. 게이트웨이** | **[컴플라이언스 게이트웨이 작동] The Gateway Intercept** | 🛡️ **중앙 장치:** 가장 두껍고 복잡한 구조의 '게이트' 아이콘 배치. 이 장치가 모든 데이터 흐름을 강제적으로 붙잡아 처리함을 표현. | "모든 상호작용은 우리의 컴플라이언스 게이트를 통과해야만 합니다." (The Control) |
| **III. 핵심 로직** | **[불변성 확보 프로세싱] Core Processing Engine** | ⚙️ **다중 박스 구조:** 내부에서 여러 개의 메커니즘이 순차적으로 작동하는 모습. (Nonce 검증 $\rightarrow$ Time-Stamping $\rightarrow$ Hashing) | **핵심 기술 강조:** '단순 로깅'과 '법적 증거 확보 과정'의 차이를 명확히 분리하여 설명. |
| **IV. 최종 출력** | **[불변 감사 기록 생성] Immutable Audit Record (The Output)** | ✅ **최종 박스:** 가장 깔끔하고 구조화된 형태로, 최종 결과물이 담기는 영역. 배경색을 안전한 색상(Safe Color)으로 변경. | "시간과 위·변조가 불가능하게 봉인된, 법정에서 반박 불가한 증거 자산입니다." (The Assurance) |

#### 📝 **3. 구체적인 텍스트/코드 블록 (Pseudo-Code & Copy)**

**[제목 및 부제]**
*   **Title:** The Compliance Gateway: 모든 상호작용을 법적 증거로 변환하는 아키텍처.
*   **Subtitle:** *Compliance Failure Cost Avoidance Platform*

**[단계 III. 핵심 로직 상세 (Mini ROI와의 연계)]**
(이 섹션은 기술적 깊이를 보여주면서도, 최종적으로 '재무적 손실액'과 연결되어야 함)

1.  **Input Data:** `User ID`, `Action Type` (예: 데이터 접근 시도).
2.  **Gateway Check 1: Nonce & Scope Validation:** 요청의 고유 식별자(Nonce)와 권한 범위(JWT Scope)를 검증하여 위변조 가능성을 사전에 차단합니다.
3.  **Gateway Check 2: Temporal Stamping (Time-Sync):** 글로벌 표준 시간 동기화 및 타임스탬프를 강제 기록합니다. (시간 조작 불가능)
4.  **Gateway Check 3: Hashing & Chaining:** 해당 모든 메타데이터와 원본 데이터를 암호화 해시(SHA-256) 처리하고, 직전 레코드의 해시 값을 연결하여 **블록체인 방식의 무결성 사슬(Integrity Chain)**을 완성합니다.

**[최종 출력 (The Audit Record)]**
*   **레이아웃:** 3개의 명확한 필드로 구성된 테이블/카드 형태.
    1.  `Event ID`: [UUID] - 고유 식별자
    2.  `Timestamp`: YYYY-MM-DD HH:MM:SS UTC (Non-Repudiable)
    3.  `Evidence Hash`: `0xABC...DEF` (The Immutable Proof)
