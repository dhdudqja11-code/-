## 📐 Mini ROI 리스크 진단 결과 화면 (UX/UI Design Spec)

**목표:** 사용자에게 현재의 상황이 위험하며, 시스템 도입만이 유일하고 논리적인 해결책임을 강력하게 인지시킨다.
**톤앤매너:** 공학적(Engineering), 권위적(Authoritative), 간결함(Minimalism).
**핵심 원칙:** 숫자를 제시하는 것이 아니라, **'숫자가 의미하는 재앙적 시나리오'**를 보여준다.

### 🎨 1. 디자인 시스템 및 가이드라인 업데이트

| 요소 | 스펙 | 설명 및 목적 |
| :--- | :--- | :--- |
| **메인 컬러 (Trust)** | `#0B2447` (딥 네이비) | 배경, 헤더, 주요 CTA에 사용. 깊고 전문적인 신뢰감을 부여. |
| **액센트 컬러 (Risk/Warning)** | `#CC3F3F` (브릭 레드) | 위험 지표(Critical Risk), 부정적 추세선, 경고 아이콘에만 사용. 공포심 유발. |
| **성공/권장 색상** | `#1E8449` (딥 그린) | 권장되는 해결책, 긍정적 추세선, '안전' 메시지에 사용. 안도감 부여. |
| **폰트 패밀리** | Pretendard / Inter (산세리프) | 가독성이 높고 모던하며 기술적인 느낌을 주는 서체. |
| **아이콘 세트** | Feather Icons (라인 기반) | 복잡한 기호 대신, 간결하고 전문적인 '경고', '책임', '분석' 등의 의미를 전달하는 아이콘 사용. |

### 📊 2. 와이어프레임 구조 및 섹션별 상세 스펙

#### **[A] 최상단 요약 대시보드 (The Punch)**
*   **목표:** 페이지 로딩과 동시에 사용자에게 '위험'을 인지시킨다.
*   **레이아웃:** 가장 큰 타이틀 아래, 3가지 핵심 지표를 카드 형태로 배치.

| 요소 | 스펙 | 내용/가이드라인 |
| :--- | :--- | :--- |
| **메인 헤드라인** | `[클라이언트명]의 현행 시스템 리스크 분석 보고서` | 제목은 '보고서'라는 단어를 사용하여 공신력 부여. |
| **총 위험 수준 (Risk Level)** | **위험도 지수: 8.2 / 10.0** <br> *(레벨 명칭)* | **[MUST]** 시뮬레이션 결과 기반으로 동적으로 변경됨. |
| **색상 코드/경고 아이콘** | 배경 바: `#CC3F3F` (브릭 레드). 아이콘: ⚠️<br>**등급:** CRITICAL (위험 등급) | *가이드:* 색상은 단순한 경고를 넘어 '🚨 법적 대응 필요'와 같은 문구로 확장하여 위기감을 증폭. |
| **핵심 요약 지표** | **예상 미회피 손실액:** <span style="color:#CC3F3F;">₩450,000,000</span><br>**주요 리스크 영역:** 법적 컴플라이언스 (Compliance) | 숫자는 가장 크고, 색상은 위험색을 사용하며, 근거가 되는 단어(법적/재무적)를 굵게 처리. |

#### **[B] 시각화 그래프 섹션: '시간의 무게' 증명**
*   **목표:** 손실액이 현재 시스템의 문제가 아니라, *개선하지 않음으로써 발생하는 시간 지연 리스크*임을 과학적으로 증명한다. (가장 중요한 섹션)
*   **그래프 타입:** Area Chart + Line Graph 조합.

| 요소 | 스펙 | 가이드라인 및 데이터 해석 |
| :--- | :--- | :--- |
| **Y축** | 누적 예상 손실액 (단위: ₩) | 축의 시작점(0원)이 명확해야 함. |
| **X축** | 시간 흐름 (월별/분기별) | 과거 데이터(Historical Data)와 미래 예측 구간을 구분하여 표시. |
| **라인 1: 현재 추세 (The Regret Curve)** | 선 색상: `#CC3F3F` (브릭 레드). 형태: 가파른 하락 곡선. | *라벨:* "현행 운영 방식 유지 시 예상 손실 누적액" <br>*시각 효과:* 이 라인은 **'우리가 통제할 수 없는 흐름'**처럼 보이게 처리. |
| **라인 2: 최적 추세 (The Solution Curve)** | 선 색상: `#1E8449` (딥 그린). 형태: 완만한 상승 곡선 또는 평탄화된 안전 지대. | *라벨:* "본 시스템 도입 후 예상 손실 최소화액" <br>*시각 효과:* 이 라인은 **'우리가 도달할 수 있는 목표점'**처럼 보이게 처리. |
| **핵심 비교 영역 (Gap)** | 배경에 투명한 그레이 쉐이드를 깔고, 두 곡선 사이의 면적을 채움. | *캡션:* "최소 확보 가능 이익(Minimum Recoverable Value)" <br>여기가 바로 우리가 얻게 될 '금액'이라는 점을 강조. |

#### **[C] 리스크 세부 분석 및 원인 규명 (The Proof Points)**
*   **목표:** 단순히 손실액이 높다고 말하는 것을 넘어, *왜* 위험한지 구체적이고 전문적인 근거(법률/기술)를 제시한다.
*   **레이아웃:** 플립카드 또는 카운터 형식의 카드 3~5개 배치.

| 요소 | 스펙 | 가이드라인 및 콘텐츠 예시 |
| :--- | :--- | :--- |
| **리스크 제목** | `[Icon] 미확인 데이터 주권 침해 가능성` | 기술적 용어와 비즈니스 리스크를 결합하여 전문성을 높임. |
| **영향도 점수 (Impact Score)** | 9/10점 (위험색 배경) | 이 수치는 '누가 계산했는지'에 대한 신뢰도를 보여주기 위해, 작은 출처 표기(e.g., *Audit Log v3.2*)를 넣어야 함. |
| **발생 원인 (Root Cause)** | "현재 사용되는 OAuth 구현은 Rate Limiting 및 Context-Aware JWT Scope 검증을 누락하여 외부 우회에 취약합니다." | 코다리나 이전 세션의 기술 용어를 활용하여, 고객이 '우리도 이런 걸 모를까?'라는 생각에 빠지게 유도. |
| **법적 근거 (Authority)** | "GDPR Article 32 위반 소지가 있습니다." | 추상적인 위험을 구체적인 법 조항으로 연결시켜 권위 극대화. |

#### **[D] 다음 단계: 액션 플랜 및 CTA (The Mandate)**
*   **목표:** 고민할 시간을 주지 않고, '지금 당장' 이 시스템에 가입하거나 상담하게 만든다.
*   **레이아웃:** 명확한 3단계 프로세스로 제시.

| 요소 | 스펙 | 가이드라인 및 CTA 구조 |
| :--- | :--- | :--- |
| **헤드라인** | `[클라이언트명]의 리스크를 제거하기 위한 3단계 필수 조치` | 명령조(Mandatory) 사용. 선택지가 아님을 암시. |
| **Phase 1: 진단 및 감사 (Audit)** | **액션:** 전용 게이트웨이 API 테스트 권한 요청 <br>**결과:** 정확한 리스크 레벨 확정. | *버튼:* [무료 리스크 진단 시작하기] |
| **Phase 2: 방어 시스템 구축 (Shielding)** | **액션:** 핵심 컴플라이언스 모듈 설계 및 통합 (Mini ROI 시뮬레이터 로직 적용) <br>**결과:** 법적 증거 확보 가능한 데이터 구조. | *버튼:* [맞춤 컨설팅 요청] |
| **Phase 3: 지속적 관리 (Guardian)** | **액션:** 전용 SaaS 플랫폼 구독 및 실시간 감사 로그 시스템 운영 <br>**결과:** 영구적인 사업 연속성 보장. | CTA는 가장 눈에 띄게, 그리고 '지금'을 강조. |
