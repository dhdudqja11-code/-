# 📋 작업 브리프

**원 명령:** [자율 사이클 — 2026-05-20] 1인 기업 24시간 운영 중. 회사 목표·각 에이전트의 개인 목표(_agents/{id}/goal.md)·최근 의사결정·메모리를 검토해서 지금 가장 가치 있는 단일 작업 1개를 결정하고, 적절한 1~2명 에이전트에게 분배해서 실행하세요. 같은 산출물을 반복하지 마세요 — 메모리에 비슷한 항목이 24시간 내에 있으면 다른 각도로 진전시키세요.

## 요약
현재 핵심 과제는 'Avoided Loss' 시스템을 디자인 단계에서 실제 개발 및 콘텐츠화하는 것입니다. 다음 자율 사이클에서는, 가장 복잡한 변수들을 코딩 가능한 API 스켈레톤으로 구현하고(Developer), 이를 사용자에게 전달할 논리적이고 권위적인 카피라이팅 기반을 동시에 다지는 것이 목표입니다.

## 분배
- **💻 코다리**: 최근 디자인 브리프에서 정의된 'Emotional Loss' 및 행동경제학 변수(예: 매몰 비용 함정 규모)를 포함하는 통합 API 스켈레톤을 작성하십시오. 특히, 이 복합적인 변수를 입력받아 최종 Avoided Loss 값을 계산할 수 있는 POST 엔드포인트의 기본 구조(FastAPI/Pydantic 모델)와 최소한의 유닛 테스트 케이스를 구현하고 그 결과를 보고하세요.
