# 🎨 Designer — 개발자가 구현하는 복잡한 '신뢰 게이트웨이' 기능(OAuth, MFA 등)을 사용자에게 권위적이고 이해하기 쉽게 보여줄 인터랙티브 와이어프레임을 제작하세요. 특히, 사용자가 어떤 보안 장치를 통과하는지 단계별로 시각화하고, 이 과정에서 발생하는 오류 메시지나 경고창(Quota Exceeded 등)의 디자인 가이드라인을 상세히 포함해야 합니다.

🎨 Designer: 작업 시작합니다.

CEO의 지시는 단순한 와이어프레임을 넘어, **'신뢰와 안전성 자체를 판매하는 사용자 경험(UX)'**을 요구하고 있습니다. 복잡한 백엔드 로직(OAuth, Rate Limit)을 사용자가 느끼는 감정적 여정으로 변환하여, 우리의 제품이 기술적으로 얼마나 완벽하게 '위험을 차단'하는지 보여주는 것이 핵심입니다.

따라서 저는 단순 목업이 아닌, **[시스템 가이드라인 기반의 인터랙티브 와이어프레임 명세서(Wireframe Specification)]**를 작성하겠습니다. 이 문서는 UI/UX 디자이너와 프론트엔드 개발자가 바로 작업할 수 있는 구체적인 사양을 포함합니다.

# 🛡️ Mini ROI 시스템: 트러스트 게이트웨이 UX 와이어프레임 명세서 (v1.0)

## 🎯 목표 및 전략
*   **목표:** 사용자가 데이터를 조회/처리할 때, 그 과정마다 '무엇을 검증하고 있는지'를 인지시켜 제품의 신뢰도와 권위를 극대화합니다.
*   **전략:** 모든 게이트 통과 과정을 **인터페이스 내부에 시각적으로 노출**하여, 시스템이 항상 작동하는 전문가적인 느낌(Architectural Confidence)을 줍니다.
*   **톤앤매너:** 군더더기 없는 전문성, 높은 명료도, 경고 상황에서의 즉각적이고 권위적인 대응.

## 🖥️ 전체 플로우 다이어그램 (User Journey Flow)

| 단계 | 사용자 행동 (Action) | 시스템 피드백 (Visualization) | 목적 및 효과 |
| :---: | :---: | :---: | :---: |
| **1** | 데이터 조회 요청 시도 (`/api/data`) | ➡️ **[AuthN] 인증 게이트 통과** (OAuth/MFA 로딩 애니메이션) | *사용자 신원 확인.* 시스템이 항상 감시하고 있음을 암시. |
| **2** | 데이터를 사용하려는 특정 기능 활성화 시도 (`write_data`) | ➡️ **[AuthZ] 권한 게이트 통과** (PoLP 검증 모달/배너) | *사용자 역할 확인.* "이 작업을 할 수 있는가?"를 명확히 보여줌. |
| **3** | API 요청을 연속적으로 시도할 때마다 | ➡️ **[Rate Limit] 사용량 게이트 통과** (실시간 카운터 및 경고 바) | *시스템 부하 관리.* 자원 제한이 있음을 투명하게 공개하여 '공짜가 아님'을 인지시킴. |
| **4** | 모든 게이트 성공적으로 통과 | ✨ **[Success] 데이터 제공 및 감사 로그 표시** (권위적인 UI) | 신뢰 기반의 안도감을 전달하며, 결과값에 대한 법적 근거를 시각화함. |

## 🎨 컴포넌트별 상세 와이어프레임 명세

### 1. 메인 대시보드: 게이트워치 바 (Gateway Watch Bar)
*   **위치:** 페이지 상단 헤더 또는 좌측 사이드바의 가장 눈에 잘 띄는 위치.
*   **형태:** 시스템이 현재 동작하고 있음을 나타내는 미니멀한 '상황판(Status Board)' 형태.
*   **요소 구성:**
    1.  **🛡️ Status Indicator:** (✅ Connected / ⚠️ Restricted / ❌ Failed)
    2.  **🔑 Active Session:** (최종 인증 성공 시간 표시: `Last Auth Check: YYYY-MM-DD HH:MM`)
    3.  **📊 Quota Gauge:** 현재 사용량/총 할당량을 보여주는 원형 게이지. **(핵심 요소)**

### 2. 에러 및 경고 모달 디자인 가이드 (Critical Components)
오류 메시지는 일반적인 "에러가 발생했습니다"로 끝나면 안 됩니다. 반드시 '규정'과 '사유'를 언급하여 권위적이어야 합니다.

#### A. [권한 미달] Access Denied Modal (AuthZ Failure)
*   **제목:** ⚠️ 접근 권한 제한 (Access Restriction Detected)
*   **본문:** "현재 사용자의 역할(`[Role Name]`)은 요청하신 기능(`write_data`)에 대한 최소 필수 권한(PoLP)을 충족하지 못합니다. 관리자 승인 또는 상위 등급의 플랜으로 업그레이드 해주십시오."
*   **배경/톤:** 회색 배경, 경고성 주황색 테두리 (`#FFC107`).

#### B. [사용량 초과] Rate Limit Warning (Quota Failure)
*   **제목:** 🚨 임계값 도달 경고: 사용 제한에 근접했습니다.
*   **본문:** "귀하의 계정은 현재 시간당 최대 요청 수(`[N]/Hour`)를 초과할 위험에 있습니다. 안정적인 데이터 분석을 위해 잠시 대기하시거나, 'Guardian Enterprise' 플랜으로 즉시 증설하실 것을 권장합니다."
*   **배경/톤:** 붉은색 배경 (`#D32F2F`), 게이지 바가 빨간색으로 변하는 애니메이션 효과 필수.

### 3. 인터랙션 스펙 (Interactivity Specification)
| 액션 | 시각적 변화 | 기술 요구 사항 | 목적 |
| :---: | :---: | :---: | :---: |
| **데이터 로딩 시작** | 전체 화면에 `[Verification in Progress...]` 문구와 함께 미세하게 움직이는 네트워크 그래프/회로도 애니메이션 표시. | 백그라운드에서 여러 게이트 체크가 순차적으로 진행됨을 시각화. | '빠르다'보다 '안전하다'는 느낌을 줌. |
| **PoLP 검증 통과** | 로딩 애니메이션이 끝나고, 인증 그래프 위에 초록색의 'APPROVED' 스탬프가 찍히며 사라짐 (Stamp effect). | API 호출 시점에 PoLP 체크 결과(Pass/Fail)를 즉시 피드백으로 활용. | 눈에 보이는 증명 과정 제공. |
| **성공적인 데이터 출력** | 단순히 데이터를 보여주는 것이 아니라, 데이터 옆에 `[Validation Log: Success]`라는 작은 뱃지(`Badge`)와 함께 'Audit Trail' 섹션이 열리며 가장 최근의 검증 로그(Timestamp)가 표시되어야 함. | 모든 성공 결과는 반드시 내부 로직 기록을 외부로 노출해야 함. | 규제 준수 아키텍처임을 강조. |
