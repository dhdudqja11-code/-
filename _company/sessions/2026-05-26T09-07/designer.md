# 📐 Mini ROI 진단 결과 페이지: 디자인 시스템 및 컴포넌트 라이브러리 스펙 (v1.0)

**목표:** 사용자가 자신의 위험 수준을 '감정적 위협'이 아닌, '객관적인 재무적 리스크(ALV)'와 '법적 근거'를 통해 인지하게 한다.
**톤앤매너:** 권위적, 냉철함 (Authoritative, Clinical). 감성적 공포 조성보다 통제된 전문가의 분석 결과를 제시하는 느낌.

## 1. 글로벌 스타일 가이드 (Global Style Guide)

| 요소 | 상세 스펙 | 설명 및 사용 원칙 |
| :--- | :--- | :--- |
| **주요 색상 (Primary)** | `#0A2342` (딥 네이비 블루) | 배경, 섹션 구분선, 주요 헤드라인에 사용. 깊고 신뢰감을 주는 색상. |
| **보조 색상 (Secondary)** | `#6C7D98` (뮤트 그레이/스틸) | 텍스트 본문, 비활성화 요소(Disabled State), 부가 정보 설명에 사용. |
| **강조 색상 (Accent)** | `#E31B23` (위험 경고 레드) | 위험 레벨 게이지의 Critical 상태 및 '즉시 조치 필요' CTA 버튼에만 제한적으로 사용. |
| **성공/안도 색상** | `#4CAF50` (세이프 그린) | Low Risk 상태 또는 시스템 개입 성공 후 '완화 완료' 표시 등에 한하여 보조적으로 사용. |
| **기본 폰트 (Typeface)** | Inter, Noto Sans KR (가장 높은 가독성을 가진 산세리프 계열 권장) | 기술 보고서나 UI에 최적화되어 신뢰감을 높임. |

### Typography Scale (타이포그래피 스케일)
*   **H1 (페이지 제목):** 32px / Bold / `#0A2342`
*   **H2 (섹션 제목):** 24px / Semi-Bold / `#0A2342`
*   **Body Large:** 18px / Regular / `#333333`
*   **Body Default:** 16px / Regular / `#555555`
*   **Caption/Code:** 12px / Mono-spaced (monospace) / `#777777`

## 2. 핵심 컴포넌트 상세 스펙 (Core Components Specification)

### A. 위험 수준 게이지 (Risk Level Gauge - 가장 중요)

이 컴포넌트는 사용자의 상태를 단번에 인지시키는 페이지의 시각적 중심입니다.
*   **형식:** 가로 막대(Horizontal Bar Chart) 형태.
*   **크기:** Width: 100% / Height: 35px. (모든 레벨에 통일 적용)

| 위험 레벨 | 색상 코드 (HEX) | 상태 설명 | 시각적 강조 지침 |
| :--- | :--- | :--- | :--- |
| **Critical** | `#E31B23` | 즉각적인 법률/재무 리스크 존재. 시스템 개입 필수. | 게이지 전체가 깜빡이거나 (Pulse Animation), 경고 문구(`CRITICAL ALERT`)를 굵게 표시해야 함. |
| **High** | `#FFC300` | 높은 위험도 감지. 즉각적인 검토 및 완화책 필요. | 주황색을 사용하되, 배경에 그림자(Shadow) 효과를 넣어 시급성을 부여. |
| **Medium** | `#FFAA44` | 주의 단계. 몇 가지 취약점 존재. 정기적 모니터링 권장. | 표준적인 경고 색상으로, 정보성 톤을 유지. |
| **Low** | `#6B9D7F` | 위험 요소 낮음. 시스템이 완벽하게 통제되고 있음을 의미. | 녹색 계열의 안정감을 주며, '안도감' 위주의 커뮤니케이션에 사용. |

### B. 예상 손실액 (ALV) 데이터 카드

정량적 재무 정보를 제시하여 감성적인 위험을 이성적인 리스크로 전환시키는 핵심 요소입니다.
*   **레이아웃:** 3-Column Grid Layout 권장. 가장 왼쪽 컬럼에 ALV가 위치하며, 시선이 자연스럽게 흐르도록 설계해야 합니다.
*   **컴포넌트 구조 (ALV Card):**
    1.  **Headline Value:** `($XX Million)` / **폰트**: Body Large (24px) / **굵기**: Bold / `#E31B23` (Critical 시). 이 숫자가 페이지의 가장 큰 요소여야 합니다.
    2.  **Label:** "최소 예상 재무 손실액 (Min. Estimated Loss Amount)" / 폰트: Body Default (16px) / 색상: `#555555`.
    3.  **Explanation/Subtitle:** "이는 규제 위반 및 데이터 유출로 인해 발생할 수 있는 최소한의 비용 추정치입니다." / 폰트: Caption (12px) / 색상: `#777777`.

### C. 법률 근거 제시 섹션 (Legal Basis & Provenance Block)

이 블록은 시스템의 신뢰도와 권위성을 극대화합니다. 단순한 '정보'가 아니라, '근거'임을 보여줘야 합니다.
*   **레이아웃:** 마치 전문적인 학술지나 법률 문서에서 발췌된 것처럼 보이도록 디자인해야 합니다 (Quotation Block 스타일).
*   **요소 구성:**
    1.  **출처 태그 (Source Tag):** 좌측 상단에 작은 박스로 `[법적 근거: GDPR Article 17]` 또는 `[규제 표준: SOX Compliance]`를 표시합니다. 이 태그는 반드시 회사의 고유한 엠블럼(Mini ROI의 로고)과 함께 배치되어야 합니다.
    2.  **본문 (Article Text):** Body Default (16px), `#333333`. 글자 간격을 미세하게 넓혀 전문적인 느낌을 강조합니다.
    3.  **요약 해석 (Our Analysis):** 본문의 하단에 별도의 옅은 배경색 박스(`Background: #F0F5FA`)를 두고, "당사의 시스템이 이 규정 위반 리스크를 어떻게 완화하는지 분석한 결과..."와 같은 권위적인 해석을 제공합니다.
