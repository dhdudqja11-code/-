# 🎨 Designer — Developer가 제작할 핵심 모듈의 프론트엔드 목업을 염두에 두고, 사용자가 필요한 변수(Pain factors)를 입력하고 시뮬레이션 결과를 확인할 수 있는 '데이터 입력 및 결과 확인 플로우' 와이어프레임 세트를 제작해 주세요. 전문적이고 신뢰감 높은 브랜드 톤과 함께, 정보의 흐름이 논리적으로 느껴지도록 배치하는 것이 중요합니다.

🎨 Designer: 작업 시작합니다.

CEO님의 지시와 목표를 종합적으로 검토했습니다. 현재까지의 논리적 흐름은 **'규제 위협 및 명성 손실 $\rightarrow$ 측정 가능한 재무적 위험 회피 비용(Avoided Loss) 증명 $\rightarrow$ 시스템 도입 확신'** 입니다.

이 핵심 프로세스를 가장 직관적으로 체험하게 하는 것은, 사용자가 직접 변수를 입력하고 그 결과가 실시간으로 '위험 감소'라는 형태로 시각화되는 **시뮬레이터 페이지**입니다. 단순히 결과를 보여주는 것을 넘어, 이 과정을 통해 서비스의 가치를 체감하도록 설계하는 것이 핵심 목표입니다.

전문적이고 신뢰감 높은 브랜드 톤을 유지하며, 정보 흐름이 논리적으로 느껴지는 '데이터 입력 및 결과 확인 플로우' 와이어프레임 세트 3종을 제안합니다. 이 구조는 개발팀(코다리)이 곧바로 프론트엔드 컴포넌트로 변환할 수 있도록 모듈화되어 있습니다.

## ⚙️ Wireframe Set: 데이터 입력 및 시뮬레이션 플로우 (3 View)

### 🖼️ View 1: 리스크 진단 - Pain Point 유도 페이지 (The Trigger)
**목표:** 사용자가 스스로 '위험'을 인식하게 하여, 우리 시스템의 필요성을 자각시키는 단계.
**컨셉:** 질문 기반의 인터페이스(Quiz/Diagnostic).

| 요소 | 설명 및 상세 디자인 지침 | 기술적 구현 가이드 |
| :--- | :--- | :--- |
| **제목 영역 (Hero)** | **"귀사의 현재 운영 모델은 몇 %의 리스크를 안고 있습니까?"** (강렬한 질문) <br> 서브 텍스트: "규제 변화와 시장 변동성이 야기하는 잠재적 재무 손실을 객관적으로 진단합니다." | 큰 타이포그래피, 배경에 미세한 노이즈 또는 경보등 패턴 사용. |
| **진단 질문 블록** | 3~5개의 핵심 리스크 카드를 제시 (예: "최근 1년간 규제 변화 대응 시간?", "주요 파트너사의 평판 변동성 체감도?"). <br> 각 질문은 드롭다운 또는 슬라이더를 사용하며, 답변이 낮을수록 '위험 점수(Risk Score)'가 높아지도록 설계. | `InputComponent` (선택형/슬라이더). 답변에 따라 실시간으로 *총 리스크 점수*가 계산되어 상단에 표시됨. |
| **결과 유도 CTA** | **"진단 결과 보고서 받기 및 상세 시뮬레이션 실행"** 버튼. 이 버튼을 누르면 View 2로 이동하며, 사용자가 입력한 데이터가 다음 단계의 기본값으로 전달되도록 설정. | Primary Button (강조색). |
| **톤앤매너** | 질문 중심적이고 위압적인 신뢰성. '불확실성'을 시각적으로 표현할 것 (흐릿함, 경고색 사용). |

### 📊 View 2: 데이터 입력 및 변수 설정 페이지 (The Engine)
**목표:** 추상적인 위험을 구체적인 수치로 치환하고, 시스템의 작동 원리를 투명하게 보여주는 단계.
**컨셉:** 전문 연구소 보고서 형태의 대시보드.

| 요소 | 설명 및 상세 디자인 지침 | 기술적 구현 가이드 |
| :--- | :--- | :--- |
| **섹션 제목** | **"위험 요인별 재무 영향 변수 설정 (Factor Input)"** <br> 서브 텍스트: "각 리스크 요소($C_{reg}, C_{rep}$)가 회사 매출 및 운영 자본에 미치는 가중치를 구체적으로 입력해 주세요." | 전문적인 섹션 구분선, 데이터 테이블 형태의 레이아웃. |
| **핵심 변수 입력 필드** | **1. 규제 리스크 비용 ($C_{reg}$):** (예: 예상되는 법적 벌금 규모 / 대응에 필요한 내부 인력 투입 비용). <br> **2. 명성 손실 비용 ($C_{rep}$):** (예: 평균 고객 이탈률 증가분 * 연간 매출액). <br> **3. 운영 지연 리스크:** (시간 단위의 가용 자산 감소율). | `InputComponent` (숫자 입력, 통화 포맷 적용 필수). 각 필드 옆에 해당 변수가 무엇을 의미하는지 설명하는 툴팁(Tooltip) 제공. |
| **시뮬레이션 실행 버튼** | **"회피 손실액($\text{Avoided Loss}$) 계산 및 시뮬레이션 시작"**. 이 과정은 로딩 애니메이션과 함께 '고급 분석 중...' 메시지를 표시해야 함. | Primary Button (강조색). 클릭 후 View 3으로 자동 전환되도록 설계. |
| **톤앤매너** | 정밀함, 투명성. 데이터가 수식이나 공식에 의해 계산된다는 느낌을 주어야 함 (공식을 작게 배치). |

### ✅ View 3: 시뮬레이션 결과 대시보드 (The Assurance)
**목표:** 입력된 변수를 기반으로 산출된 '회피 손실액'을 가장 강력한 설득력으로 제시하는 단계. (Coda-ri가 설계한 컴포넌트 활용).
**컨셉:** 극적인 변화(Pain $\rightarrow$ Relief)의 시각화.

| 요소 | 설명 및 상세 디자인 지침 | 기술적 구현 가이드 |
| :--- | :--- | :--- |
| **핵심 결과 요약 (Top)** | **"총 회피 손실액: $XXX,ZZZ,LLL"** 이 수치를 가장 크게 중앙에 배치. <br> 부제목: "이 금액은 귀사가 잠재적인 규제 및 평판 위협으로부터 확보한 최소 보장 자산입니다." | Mega-Typography (가장 큰 폰트 크기). 배경 색상이나 애니메이션으로 '안전함'을 시각화 (예: 녹색 계열의 빛). |
| **결과 분석 섹션** | **[Avoided Loss Impact Card]** 컴포넌트를 배치. 이 컴포넌트는 View 2에서 입력된 변수들이 어떻게 조합되어 이 최종 금액을 만들어냈는지, 그 로직적 경로를 시각적으로 보여줘야 합니다. (Coda-ri가 작성한 React 컴포넌트가 들어갈 자리). | `RiskImpactCard` 컴포넌트 사용. 데이터는 API 응답 값으로 바인딩됨. |
| **추가 증명 요소** | 3~4개의 '통제력 확보 요인'을 리스트업 (예: "선행 규제 감지 시스템 도입", "AI 기반 평판 모니터링"). 각 요인은 아이콘과 간결한 설명으로 배치되어, 이 결과가 단순 계산이 아님을 증명해야 합니다. | Iconography + Bullet Point List. |
| **CTA** | **"전체 리스크 분석 보고서 다운로드 (PDF)"** 버튼. 최종 구매/상담 단계로 유도합니다. | Secondary Button. |
